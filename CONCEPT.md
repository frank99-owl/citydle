# Citydle — 产品设定（Concept）

> 本文件是新方向的「北极星」。当前仓库代码仍是旧玩法（拼写金融街道名），本文件描述的是**将要重构成的目标**。已实现 vs 计划，下面明确区分。

中文名沿用 **每日街图**。

---

## 一句话

每天一座城市，只给你它**真实的地图骨架**（路网 + 地理特征），从最抽象的剪影开始逐层揭示，**6 条线索内、6 选 1** 认出它是哪儿。一个「读地图认城市」的每日推理小游戏。

## 核心玩法

- **每日一题，全球同题**（像 Wordle，制造「今天你猜了吗」的同步感）
- **6 层线索递进**（从抽象到具体）：
  1. 主干道剪影
  2. + 主路网 / 水系（海岸线、河流）
  3. + 完整路网纹理 ← 信息量最大、形状个性最强的一层
  4. + 地标位置
  5. + 一条街名
  6. + 国家 / 首字母（兜底，不直接给答案）
- **6 选 1**（候选城市）：降低门槛，让猜城市从「凭空打字」变成「人人能上手」。难度做在「**你用了几条线索**」，不是「前几条几乎空白」。
- 用**越少线索**认出 → 分越高。
- **🟩🟥⬜ 方块分享**（Wordle 式病毒钩子）+ **streak**（连续天数）。

## 世界观 / 美学

- **沧桑制图师**：羊皮纸、暖金线条地图。你在读一张张「老地图残片」，认出它画的是哪座城。
- 这让「猜城市」从冷冰冰的测验，变成**有审美、有情绪**的体验。
- **天然差异化**：GeoGuessr 用照片 / 街景，Citydle 用**线条地图本身**。「读图认城」是独占定位，不跟它正面打。也承接了原项目名 `cartographer`（制图师）的灵魂。

## 数据原则（底线，不可妥协）

游戏里你看到的每一根线，都是那座城市地面上**真实存在**的路。

- 路网 → Overpass API（OSM）
- 海岸线 / 河流 / 水体 → OSM `natural=coastline`、`natural=water`、`waterway`
- 地标 → OSM POI（`tourism` / `amenity` 等）；**拉不到就不放，绝不编造**
- 每座城市数据带**来源标记**（`source: overpass` + 拉取日期），可追溯
- **零手工捏造坐标**
- 管线同时拉 way 与 **multipolygon relation 水体**的成员 way（悉尼港这类海湾湖泊不会缺失）

## 城市库

- **已扩至 100 个高辨识度世界城市**（每日一题一轮 100 天不重复；每城带国家元数据）
- **选片原则**：有海岸 / 有河 / 路网形态独特（网格、放射状、不规则老城）；**避开「哪座城都长一样」的纯住宅郊区**——那种没法猜。
- **生成方式**：批量 Overpass pipeline（复用现有 `api/streets` 引擎），人工挑城市 + 代表性区域，机器拉数据。

## 与旧版（Financial Street Cartographer）的区别

| | 旧版 | Citydle |
|---|---|---|
| 题材 | 5 个金融街区 | 世界城市（不限金融） |
| 玩法 | 在地图上**拼写街道名** | **认出是哪座城市**（6 选 1） |
| 门槛 | 硬核记忆，受众极窄 | 推理，大众可玩 |
| 复用 | — | 地图引擎、Overpass、沧桑美学、每日生成、i18n |
| 废弃 | — | 拼街道核心、11 成就、自定义画区、旧排行榜 |
